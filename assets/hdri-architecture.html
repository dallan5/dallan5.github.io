Pix2Pix: Backplate → HDRI

Complete architecture and training flow

1. The Big Picture — What Happens Each Training Step

Generator (U-Net)

Takes a 128×128 backplate photo and tries to paint a full 128×256 equirectangular HDRI from it.

Goal: fool the discriminator into thinking the output is a real HDRI, while also matching the ground truth pixel-by-pixel.

128×128×3 → 128×256×3

fake HDRI
→
←
"real or fake?"

Discriminator (PatchGAN)

Sees the backplate + an HDRI (either real or generated) and judges whether each local patch looks real or fake.

Goal: correctly identify which HDRIs are real ground truth and which are generator fakes.

128×256×6 → 16×33×1

The GAN game: The generator and discriminator compete. The generator gets better at making realistic HDRIs because the discriminator keeps catching its mistakes. The discriminator gets better at spotting fakes because the generator keeps improving. In theory they push each other upward — in practice, the discriminator often wins (as we've seen with D loss dropping to 0.047).
2. Generator Architecture — The U-Net
1
Input Backplate
Your photo. B × 3 × 128 × 128
Pixel values normalized to [-1, 1]. Three channels: R, G, B.
2
Stretch to Target Aspect Ratio
F.interpolate(x, size=(128, 256), mode="bilinear")
Stretches the square backplate to 2:1 panoramic ratio. 128×128 → 128×256
No learning here — just bilinear interpolation. This ensures the skip connections between encoder and decoder layers have matching sizes.
3
Encoder — Compress Down (6 layers)
Each layer: Conv2d(stride=2) → InstanceNorm → LeakyReLU(0.2)

3 × 128×256 → down1 → 64 × 64×128 (d1)
64 × 64×128 → down2 → 128 × 32×64 (d2)
128 × 32×64 → down3 → 256 × 16×32 (d3)
256 × 16×32 → down4 → 512 × 8×16 (d4) + dropout 50%
512 × 8×16 → down5 → 512 × 4×8 (d5) + dropout 50%
512 × 4×8 → down6 → 512 × 2×4 (d6) — no norm

Each step halves the spatial size but increases channels. The image gets squeezed into a dense code.
Dropout on layers 4-5 forces the network to not rely on any single feature — prevents overfitting.
4
Bottleneck
512 × 2 × 4 = only 4,096 values to represent the entire scene.

This is where the network is forced to "understand" the image. It can't store pixel details here — only high-level concepts like: indoor/outdoor, time of day, overall brightness, dominant colors. Everything else was stripped away by the encoder.
5
Decoder — Expand Back Up (5 layers + final)
Each layer: ConvTranspose2d(stride=2) → InstanceNorm → ReLU → concatenate with encoder skip

512 × 2×4 → up1 → upsample to 4×8, cat with d5 → 1024 × 4×8
1024 × 4×8 → up2 → upsample to 8×16, cat with d4 → 1024 × 8×16
1024 × 8×16 → up3 → upsample to 16×32, cat with d3 → 512 × 16×32
512 × 16×32 → up4 → upsample to 32×64, cat with d2 → 256 × 32×64
256 × 32×64 → up5 → upsample to 64×128, cat with d1 → 128 × 64×128

The skip connections are the key insight. Each decoder layer gets TWO inputs: the upsampled abstract representation from below AND the fine-grained features saved from the encoder. This lets the network combine "I know this is a daytime outdoor scene" (from bottleneck) with "these exact edge pixels look like this" (from skip).
6
Final Layer → Output HDRI
ConvTranspose2d(128 → 3, stride=2) → Tanh
Upsamples from 64×128 to 3 × 128 × 256

Tanh squishes all values to [-1, 1]. This is your generated HDRI.
Why skip connections cause "input copying": The easiest thing for the network to learn is to pass the encoder features straight through to the decoder unchanged. The backplate region of the HDRI should look exactly like the input, so the skip connections happily carry that through. The challenge is getting the network to generate NEW content for the parts of the panorama that aren't in the original photo.
3. Discriminator Architecture — The PatchGAN
1
Input: Backplate + HDRI (concatenated)
The backplate (resized to 128×256) and an HDRI (real or generated) are stacked along the channel dimension.
3ch backplate + 3ch HDRI = 6 × 128 × 256

The discriminator sees BOTH images together — it's not just asking "does this HDRI look real?" but "does this HDRI make sense given THIS backplate?"
2
Instance Noise (training only)
x = x + randn_like(x) * noise_std

Random Gaussian noise added to the input before the discriminator processes it. Starts at std = 0.1, linearly decays to 0 by epoch 75.
Purpose: blurs the boundary between real and fake, preventing the discriminator from latching onto tiny pixel-level artifacts. Without this, D gets too good too fast and the generator learns nothing from the adversarial signal.
3
Convolutional Layers
6 × 128×256 → Conv(stride=2) → 64 × 64×128
64 × 64×128 → Conv(stride=2) + Norm → 128 × 32×64
128 × 32×64 → Conv(stride=2) + Norm → 256 × 16×32
256 × 16×32 → ZeroPad + Conv + Norm → 512 × 16×32
512 × 16×32 → ZeroPad + Conv → 1 × 16 × 33
4
Output: Patch Predictions
1 × 16 × 33 = 528 patches, each covering roughly a 70×70 pixel region of the original image.

Each value is a score: high = "this patch looks real", low = "this patch looks fake".
This is better than a single real/fake score because it forces the discriminator to check EVERY part of the image. The generator can't get away with making one area look good and ignoring the rest.
4. Loss Functions — What Drives Learning

Generator Loss

L_G = L_GAN + 50 × L_L1

L_GAN (adversarial): MSE between discriminator's prediction on the fake HDRI and 1.0 (what "real" looks like). The generator wants the discriminator to output all 1s for its fake images.

L_L1 (reconstruction): Average absolute pixel difference between generated HDRI and ground truth. Forces the output to be close to the correct answer.

λ = 50: L1 is weighted 50× more than GAN loss. This means pixel accuracy dominates — which is why outputs are blurry but color-correct. The GAN loss adds sharpness but is a minority voice.

Discriminator Loss

L_D = 0.5 × (L_real + L_fake)

L_real: MSE between prediction on real HDRI and 0.9 (not 1.0 — label smoothing). Penalizes the discriminator for not recognizing real images.

L_fake: MSE between prediction on generated HDRI and 0.1 (not 0.0 — label smoothing). Penalizes for not catching fakes.

Label smoothing: Using 0.9/0.1 instead of 1.0/0.0 prevents the discriminator from becoming 100% confident, which would kill gradient flow to the generator.

LSGAN (Least Squares): Both losses use MSE instead of the traditional binary cross-entropy. With BCE, once the discriminator is confident, gradients vanish — it stops teaching the generator. With MSE, even confident predictions still produce meaningful gradients. More stable training.
5. One Training Step — What Happens Per Batch
Step A — Train Discriminator

Freeze Generator, Update Discriminator

1. Generator makes fake HDRIs (no gradients tracked)

2. Discriminator scores real pairs:
  pred_real = D(backplate, real_hdri)

3. Discriminator scores fake pairs:
  pred_fake = D(backplate, fake_hdri)

4. Compute D loss (wants pred_real≈0.9, pred_fake≈0.1)

5. Backpropagate → update D weights only

→
Step B — Train Generator

Freeze Discriminator, Update Generator

1. Generator makes fake HDRIs again (gradients ON)

2. Discriminator scores the fakes:
  pred_fake = D(backplate, fake_hdri)

3. GAN loss: how far is pred_fake from 1.0?
  (generator wants D to say "real")

4. L1 loss: pixel difference vs ground truth

5. Total = GAN + 50×L1

6. Backpropagate through D into G → update G weights only

Key detail in Step B: The gradients flow backward through the discriminator's network into the generator. The generator learns "if I change this pixel, the discriminator's score changes by this much." The discriminator's weights don't change — it's used as a frozen lens that the generator learns to see through.
6. Full Epoch Lifecycle
1
Set Instance Noise Level
noise_std = 0.1 × max(0, 1.0 - epoch / 75)
At epoch 0: noise = 0.1. At epoch 39: noise ≈ 0.047. At epoch 75+: noise = 0.
2
Loop Over All 433 Batches
6,934 training pairs ÷ 16 per batch = 433 batches.
Each batch: load 16 backplate+HDRI pairs → run Step A (train D) → run Step B (train G) → log losses.
3
Step Learning Rate Scheduler
Epochs 1-50: LR stays at 0.0002.
Epochs 51-100: LR linearly decays to 0. Smaller steps for fine-tuning.
4
Save Samples & Checkpoint (every 3 epochs)
Run 4 validation images through the generator and save the backplate/generated/ground-truth comparison.
Save full checkpoint (all weights + optimizer state) so training can be resumed.

Bottom line: The L1 reconstruction path is learning well — the model produces color-correct, scene-appropriate panoramas. But the adversarial path (which adds sharpness and realistic detail) has collapsed because the discriminator won the arms race. The next architectural improvement (perceptual/VGG loss) would replace the discriminator's role with a pre-trained network that measures "does this look like a real image" without the unstable GAN training dynamic.
